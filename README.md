## Overview

BuildBox is aimed at solo developers and teams that prefer a visual workflow over a code-first setup. With drag-and-drop editing, visual scripting, and asset organization features, it helps streamline how game logic is assembled, dependencies are tracked, and concepts move into preview with less overhead.

The tool is built for desktop and web scenarios and supports export paths for multiple runtime environments. It is a strong fit for developers who want quick iteration, live multiplayer testing, multilingual content authoring, and AI-assisted creation within a single workspace.

## Using BuildBox

A normal workflow begins by creating a project, importing assets, and connecting game elements through the visual editor. After that, you can wire behaviors with scripting blocks, validate interactions in preview mode, and export to the destination platform once everything is ready.

Common tasks include:

- Open a project and arrange scenes with drag-and-drop tools
- Add assets and review how they connect in the dependency graph
- Use visual scripting to define gameplay behavior
- Run zero-compilation previews to iterate quickly
- Test multiplayer behavior during development
- Export builds for Windows, macOS, Linux, Android, or HTML5

## Configuration

BuildBox settings are usually handled inside the application workspace or the project files it creates. If your release includes a local config file, keep project options, export preferences, and editor behavior there instead of spreading them across multiple locations.

Example structure:

    {
      "projectName": "MyGame",
      "language": "en",
      "exportTarget": "HTML5",
      "previewMode": "zero-compilation"
    }

Adjust these values to match your workflow, target platform, and language needs.
